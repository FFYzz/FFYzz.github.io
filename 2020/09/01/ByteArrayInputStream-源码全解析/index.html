ByteArrayInputStream 源码全解析

发表于 2020-09-01 分类于 JDK 阅读次数： Disqus：

ByteArrayInputStream 与 BufferedInputStream 比较相似，两者内部都维护了一个 byte 数组类型的 buf 缓冲区。但是 ByteArrayInputStream 维护的是一个内存缓冲区(我感觉没啥作用….)。

借用一下 BufferedInputStream 中的图，其实两者是很像的。

构造函数

有构造函数可知，ByteArrayInputStream 中的初始缓冲区是由外部传进来的。同样，pos 指向下一个要读取的元素的下标，count 表示当前缓冲区一共有多少个有效数据。当传进来的 buf 是带 offset 的情况下， mark 也会被更新。

BufferedInputStream 中的 read 方法

BufferedInputStream 中一共有 4 个 read 方法。下面一一来看一下。

先判断 pos 是否为有效值。如果有效，直接返回 buf[pos]，并更新 pos。如果无效，则返回 -1。

该方法从缓冲区中读取 len 长度个字节数据，并返回实际读取的字节数。

  1. 同样先检查 pos 是否有效，不可读了则直接返回 -1。
  2. 如果可读数小于要读的数据量 len，则最多只能读取 avail 个字节数据。
  3. 将数据拷贝到传入的 b 数组中
  4. 更新 pos，返回实际读取到的数据数。

将缓冲区所有剩余可读的数据全部读取完并返回该数组。同时更新 pos。

类似于上面的 read(byte b[], int off, int len) 方法，只不过该方法不是线程安全的。

skip 方法

该方法跳过读取缓冲区中的 n 个字节数据。

  1. 首先计算剩余可读的数据量。
  2. 最多只能跳过剩余可读的个数。
  3. 返回实际 skip 的数据个数。

将缓冲区中的数据写入到 OutputStream 中

BufferedInputStream 中提供了一个方法，将缓冲区 buf 中的数据写入到 OutputStream 中。相当于 OutputStream 的功能。

其他方法

  • ByteArrayInputStream 是支持 mark 操作的
  • mark 操作会记录当前的 pos 的位置
  • reset 操作会将 pos 指向之前记录的 mark 的位置
  • available 返回缓冲区中当前可读的数据的个数
  • close 方法是一个空实现，因为 ByteArrayInputStream 是一个内存缓冲区。
