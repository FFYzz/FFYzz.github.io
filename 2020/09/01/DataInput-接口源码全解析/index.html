DataInput 接口源码全解析

发表于 2020-09-01 更新于 2020-09-02 分类于 JDK 阅读次数： Disqus：

DataInput 是一个接口，定义方法的实现规范。 DataInput 与 InputStream 不同，InputStream 读取后的数据位 byte 形式，DataInput 可以将数据读成 Primitive Java 类型和 String 类型。

下面来看 DataInput 的接口定义。

接口定义

从 input stream (与 InputStream 不同) 中读取数据存储到传入的 byte 数组中，读取的字节数为传入的字节数组的长度。若没有读满，会阻塞，直到有以下情况发生:

  • 读满正常返回
  • 读完了，但是没满，抛出 EOFException 异常。
  • IO 异常抛出 IOException

与上述方法类似，只是该读取方法指定了读取数据的存放位置为 b[off] -> b[off + len - 1]。

从 input stream 中跳过 n byte 的数据，如果跳过的字节数小于 n，那只有一个原因， input stream 的数据读完了。

从 input stream 中读取一个字节的数据，并将读到的值转为 boolean 类型的数据。

从 input stream 中读取一个字节的数据，将其转为有符号的 byte 类型数据，与 DataOutput 接口的 writeByte 方法适配。

从 input stream 中读取一个字节的数据，将其转为无符号的 byte 类型数据，与 DataOutput 接口的 writeByte 方法适配。

从 input stream 中读取两个字节的数据，转为 short 类型的值。与 DataOutput 接口的 writeShort 方法适配。
计算方法为 (short)((a << 8) | (b & 0xff))，a 为读到的第一个字节，b 为读到的第二个字节。

从 input stream 中读取两个字节的数据，转为无符号 short 类型的值。与 DataOutput 接口的 writeShort 方法适配。
计算方法为 (((a & 0xff) << 8) | (b & 0xff))，a 为读到的第一个字节，b 为读到的第二个字节。

从 input stream 中读取两个字节的数据，转为 char 类型的值。与 DataOutput 接口的 writeChar 方法适配。
计算方法为 (char)((a << 8) | (b & 0xff))，a 为读到的第一个字节，b 为读到的第二个字节。

从 input stream 中读取四个字节的数据，转为 int 类型的值。
计算方法为 (((a & 0xff) << 24) | ((b & 0xff) << 16) | ((c & 0xff) << 8) | (d & 0xff))，a-d 为依次读到的四个字节数据。

从 input stream 中读取八个字节的数据，转为 int 类型的值。
计算方法为:
(((long)(a & 0xff) << 56) |
((long)(b & 0xff) << 48) |
((long)(c & 0xff) << 40) |
((long)(d & 0xff) << 32) |
((long)(e & 0xff) << 24) |
((long)(f & 0xff) << 16) |
((long)(g & 0xff) << 8) |
((long)(h & 0xff)))
a-h 为依次读到的八个字节数据。

从 input stream 中读取四个字节的数据，转为 float 类型的值。

从 input stream 中读取八个字节的数据，转为 double 类型的值。

从 input stream 中读取一行的数据，直到遇到换行符。该方法并不支持所有的 Unicode 字符集。

读取一个用 UTF-8 编码格式进行编码的字符串。
