ByteArrayOutputStream 源码全解析

发表于 2020-09-01 分类于 JDK 阅读次数： Disqus：

ByteArrayOutputStream 与 ByteArrayInputStream 一一对应。同样也是一个面向内存的缓冲区。

ByteArrayOutputStream 中持有一个 buf 字节数组，用于充当缓冲区。 count 属性用于记录缓冲区中有效的数据量。MAX_ARRAY_SIZE 限制 buf 最大的容量。

构造方法

默认的 buf 的 size 为 32。当然也可以指定 buf 的大小。

ByteArrayOutputStream 中的 write 方法

write 方法用于向 buf 缓冲区中写入数据。

将数据 b 写入到缓冲区 buf 中，并更新 count。

将字节数组 b 的 off -> off + len 位置的数据写入到缓冲区 buf 中。更新 count。值得注意的是中间调用了 ensureCapacity 方法用于确保 buf 的容量是够的。中间涉及到缓冲区的扩容，后面展开看。

将整个数组 b 写入到缓冲区中，调用上面的 write 方法。该方法不是线程安全的。

缓冲区的扩容

首先看下 ensureCapacity 方法，该方法能确保当前缓冲区的大小是大于传入的 minCapacity 的。如果不满足，则会调用 grow 方法进行扩容。

扩容策略:

  • 将原来的缓冲区容量扩大至两倍。
  • 如果扩大至两倍后还是小于 minCapacity，则将容量指定为 minCapacity。
  • 还需要检查容量是否超出设定的 MAX_ARRAY_SIZE。如果超出了，则需要调用 hugeCapacity 方法，再次尝试扩容。这次的扩容幅度就很小了，最大只能扩大到 Integer.MAX_VALUE，也就是两者最大相差 8。
  • 最后扩容完成后进行数组拷贝。

将缓冲区中的数据写入到 OutputStream 中

该方法将缓冲区中的所有数据都写到 OutputStream 中。又想起 ByteArrayInputStream 中的一个方法，也是将缓冲区中的数据写入到 OutputStream。不过在方法命名上就更加讲究了。inputStream 中叫 transferTo，outputStream 中叫 writeTo，是比较合理的。

toString 方法

提供了一系列 toString 方法，用于将缓冲区中的数据保存成 String 类型的字符串。

其他方法

返回一个包含所有缓冲区数据的新的 byte 数组。

size 返回缓冲区中保存的数据量。

reset 方法将 count 置为 0。逻辑上的删除缓冲区中的数据，实际上并没有删除数据。

close 方法也是一个空实现，因为是一个对于内存数据的缓冲区。
