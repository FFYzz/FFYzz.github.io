BufferedInputStream 源码全解析

发表于 2020-08-30 更新于 2020-09-01 分类于 JDK 阅读次数： Disqus：

BufferedInputStream 为 InputStream 增加了新的功能。包括提供了 mark/reset 的实现等等。
BufferedInputStream 内部有一个 buf 字节数组，用于缓存从 InputStream 中读取到的数据流，每次向 Java 层缓存读取数据的时候都会一次性填满缓冲区 buf。应用层在调用 read 方法时，其实是读取缓存在 BufferedInputStream 中的 buf 数组，所以比向 Java 层缓存进行读取会快很多。

BufferedInputStream 中缓冲区的结构入下图所示，图中也给出了 BufferedInputStream 中重要的几个变量的意义。

构造方法

BufferedInputStream 提供了两个构造方法。构造方法中传入一个 InputStream。因为 BufferedInputStream 继承自 FilterInputStream，因此构造方法中传入的 in 会被赋值给父类 FilterInputStream 的成员变量 in。此外传入一个 size，用于初始化确定内部 buffer 数组。 DEFAULT_BUFFER_SIZE 的大小为 8192。BufferedInputStream 中还定义了一个 MAX_BUFFER_SIZE 为 Integer.MAX_VALUE - 8。为什么要减 8 呢？因为某些 VM 中在数组的头部会保留一个字节的 header。

BufferedInputStream 中的 read 方法

read 方法是一个同步方法

  1. 先判断缓存在 buffer 中的数据是否已经全部读完，如果读完了就调用 fill()，往 buffer 中加数据。fill() 方法是 BufferedInputStream 中的重点。
  2. 调用 getBufIfOpen() 方法检查 stream 是否关闭，未关闭返回 buf 数组，并更新 pos 。

read(byte b[], int off, int len) 方法里面的主要逻辑就是通过一个无限循环来调用 read1 方法不断进行读取，直到读到的字节数为 len。所以重点来看下 read1 方法。

  1. 首先计算得到当前 BufferedInputStream 的缓冲区 buf 中剩余可读的字节数。
    1. 如果还有剩余的字节数，则直接读取缓冲区，读取到的字节数为 剩余可读字节数 与 要求读取的字节数 中的较小者。这里不用担心读取的字节数不够。因为前面提到了，外层调用的 read 方法是一个无限循环方法，会一直读取直到读够要求的字节数，或者读取的过程中读完了。
    2. 如果剩余的没有剩余的字节数了，有两种情况。
      1. 直接调用 InputStream 的 read 方法进行读取。
      2. 调用 fill 读取 Java 层缓存到应用层缓冲区中。
  2. 进行数组拷贝，将读取的数据放入到传入的 b 数组，偏移位置从 off 开始。
  3. 方法返回该次实际读取到的字节数。

向缓冲区中写数据

向缓冲区中存数据的方法由 fill 来执行。该方法是 BufferedInputStream 类中最复杂(重要)的方法。方法里也加了比较详细的注释。

  1. 有一个前提，当调用到 fill 方法时，一定是当前的 buf 已经不可读了，换句话说， buf 中的数据都已经被读取了。
  2. 先检查是否有 mark
    1. 如果没有 mark，那么直接更新 pos 为 0， count 为 0。随后调用 InputStream 的 read 方法，将缓冲区 buf 填满。
  3. 如果 buf 数组是否已经被填满，也即没有多余的空间用于缓存
    1. 如果 mark 的位置 > 0，结合前面的情况，此时 mark 只能大于等于 0，首先处理 mark > 0 的情况。该段逻辑就是把 mark 点到后面的数据往前移。
    2. 如果 mark 的位置 = 0
      1. 并且缓冲区的容量大于 marklimit，这里的 marklimit 限制的是从 mark 位置可以读到后面的字节数。取消 mark，将 pos 指向 0，从头开始重新读取。相当于调用了一次 reset。
      2. 检查 buf 的 size，过大抛异常。
      3. 对 buf 进行扩容，扩容策略就是扩成原来的两倍。
  4. 最后一段判断之后的逻辑就是调用 InputStream 的 read 方法，从 Java 层缓冲区中读取数据。

其他方法

skip 方法，可以跳过 stream 中的指定个数字节。

  1. 当 buf 缓冲区中没有可读的数据的情况下
    1. 如果没有标记，则直接调用传入的 InputStream 的 skip 方法。
    2. 如果有标记，因为 buf 中已经没有可读数据了，所以先进行一次 fill。fill 之后更新 avail。
  2. 此时 buf 中有数据可以读了。
  3. 开始进行 skip，skip 的字节数为传入的 n 与 avail 中的较小值。
  4. 最后返回实际 skip 的字节数。

当需要一定 skip 指定的字节数的时候，可以像 read 方法一样，在外层加个无限循环。

available 方法返回一个估计值，该值是缓冲区中剩余可读的字节数加上传入的 InputStream 的 available 方法返回的结果。

上述是支持 mark 的方法实现以及 mark 方法和 reset 方法。
