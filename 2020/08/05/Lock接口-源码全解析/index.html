本文 Condition接口 源码解析基于 JDK14-GA 版本。

Condition接口 的完整源码注释可以参看本人 Github 中的 JDK14 仓库

接口方法

Lock

如果锁未被持有，那么调用 lock 方法能够获取到锁。如果锁已被其他线程持有，那么 lock 方法会被阻塞。

lockInterruptibly

该接口获取锁方法支持中断。

  • 如果调用该方法能够直接获取到锁，那么直接返回。
  • 如果未能直接获取到锁，那么将会阻塞等待。有两种情况会从阻塞状态中唤醒。
    • 获取到了锁。获取到锁能够从阻塞中唤醒的原因是同步队列中的前驱节点主动将当前线程唤醒。
    • 其他线程中断了当前阻塞的线程。

tryLock

首先该方法的调用返回两个结果，成功获取到锁或者未获取到锁。而且是立即返回。如果调用该方法的线程获取锁失败，线程不会进入同步队列等待，而是直接返回结果。

带参数的 tryLock 方法在如果在指定时间内锁被释放，那么调用线程能够成功获取到锁。如果获取锁失败，那么线程将会进入到阻塞状态，知道发生以下几种情况，才会从阻塞状态中唤醒。

  1. 线程在等待时间范围内成功获取到了锁。
  2. 在等待时间范围内被其他线程中断了，那么会响应中断，从阻塞中唤醒。
  3. 超时。

unlock

释放锁，该接口方法的实现需要检查调用线程是否就是为持有锁的线程。

newCondition

返回一个 Condition 实例。只要记住 Condition 的 await/signal 方法对标 Object 类中的 wait/notify 方法即可。
