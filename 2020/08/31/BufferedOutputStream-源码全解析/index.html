BufferedOutputStream 源码全解析

发表于 2020-08-31 更新于 2020-09-01 分类于 JDK 阅读次数： Disqus：

BufferedOutputStream 继承自 FilterInputStream，功能上与 BufferedInputStream 比较类似，都是在应用层与 Java 等缓冲区之间建立一个缓冲区。只不过 BufferedInputStream 的缓冲区方便用户一次性快速读取，避免多次向 Java 层缓冲区读取。而 BufferedOutputStream 的缓冲区则是避免用户多次向 Java 层缓冲区写数据，而是当缓冲区满之后才会向 Java 层缓冲区写一次数据。当然也是可以手动 flush 的。

构造方法

与 BufferedInputStream 类似，缓冲区的默认大小也是为 8192，同样也是可以自己指定缓冲区大小的。

BufferedOutputStream 中的 write 方法

BufferedOutputStream 中提供了两种类型的方法向缓冲区中写入数据。

  • write(int b) 方法将数据 b 写入到缓冲区中，写入之前先检查一下缓冲区是否已满，若缓冲区满，则进行一次 flushBuffer，flushBuffer 在后面会进行分析。
  • write(byte b[], int off, int len) 则是将数组 b[off] -> b[off+len] 的数据写入到缓冲区中。
    • 如果要写入的数据大于缓冲区的容量，则将缓冲区的数据进行 flush，随后调用 传入的 OutputStream 的 write 方法进行写入。
    • 否则，处理一下缓冲区，随后进行数组拷贝。

缓冲区 flush

  • flushBuffer() 方法是一个 private 方法，在进行写入的时候会被内部调用，用于将缓冲区中的数据 flush 到 Java 层缓冲区中。
  • flush() 则是先将应用层的缓冲区 flush 到 Java 层的缓冲区中，最后再调用 OutputStream 的 flush 方法，将 Java 层的缓冲区数据 flush 到底层 IO 设备中，
