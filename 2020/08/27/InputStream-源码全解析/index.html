InputStream 源码全解析

发表于 2020-08-27 更新于 2020-08-28 分类于 JDK 阅读次数： Disqus：

java.io 包下第一个要看的类就是 InputStream 抽象类。
Java 中的 IO 分为两大类，面向字节流的操作以及面向字符流的操作。分别有以下四个顶层抽象类。

本文的 InputStream 源码解析基于 JDK 11
完整的源码注释可查看 Git 仓库

InputStream 中的那些常量

首先来看下输入字节流 InputStream 抽象类中的常量定义。

BUFFER_SIZE 与读取时分配的 byte 类型数组大小有关。

  • MAX_SKIP_BUFFER_SIZE: 该值限制了每次循环中在读取输入流的时候可以跳过的最大的字节数为 2048 byte。
  • DEFAULT_BUFFER_SIZE: 设置默认每次循环中读取的最大的字节数为 8192 字节。
  • MAX_BUFFER_SIZE: 最大能够分配的数组的大小。

InputStream 中的 read 方法

read() 抽象方法

InputStream 中定义的 read 方法为抽象方法，该方法的具体实现由子类决定。

read() 方法的定义规定:

  • 该方法每次读取一个 byte 的数据。
  • 返回值为一个 [0,255] 的 int 类型的值。
  • 如果输入流读取结束，则返回 -1。
  • 该方法是一个阻塞方法，除非字节流读取完毕或者流中有数据又或者在读取的过程中抛出异常。

后续会以 BufferedInputStream 中的 read() 方法实现进行分析。

read 系列其他方法

  • 与上面的 read() 方法一致，以上两个 read 方法也是阻塞方法。read(byte b[]) 方法传入一个 byte 数组用于接收字节流数据。每次最多可以读取 byte 数组长度个字节的数据。返回值为实际接收到的字节数。其实现调用了 read(byte b[], int off, int len) 方法。注意在读取接收的数据的时候不能以 byte 数组的长度作为遍历的终点，应该以返回值作为遍历的终点。

  • read(byte b[], int off, int len) 传入一个 byte 数组以及一个偏移量 off 和读取的数据的长度 len。该方法最多读取 len 长度 byte 数据，读取到的第一个数据会存储在 b[off] 位置。

  • readAllBytes() 方法会读取所有的数据，内部调用 readNBytes(int len) 方法。该方法返回读取到的数据 byte 数组。readAllBytes() 方法设计的初衷是为了一次性读取 inputstream 中的数据，而不是读取大量的数据。如果读取的过程中发生 IO 异常，那么建议马上停止继续读取。
  • readNBytes(int len) 方法读取最多指定(len)长度个 byte。将每次读取到的结果分别保存在 buf 数组中，并加入到 List 中保存。最后通过拼接 List 中的数据，返回最终的结果。
  • readNBytes(byte[] b, int off, int len) 方法与 read(byte[], int, int) 的区别在于该方法可以处理 read 过程中被中断，或者出现 IO 异常的情况。该方法的调用会继续读取。而 read(byte[], int, int) 方法出现上述情况的时候会立刻返回。

Null Object pattern 的实现

JDK 11 之后 InputStream 中引入了一个 public static 的方法，该方法返回一个实现了 InputStream 抽象类的具体对象。该方法是一个 Null Object pattern 的实现。

skip 跳过读取数据

skip 方法提供了跳过读取过程中指定长度数据的功能。传入要跳过的数据的长度 n，返回实际跳过的数据的长度。

transferTo 方法

该方法能够将 InputSteam 中的数据快速写入到传入的 OutputStream 中。该方法有两个点需要注意:

  1. 调用该方法需要注意的是不会关闭输入流，也不会关闭输出流。所以需要手动处理输入输出流
  2. 如果读取过程或者写入过程出现 IO 异常，那么建议立刻停止。

其他方法

available() 方法由子类具体实现，返回 inputstream 中数据的长度，是一个估计值，所以不能根据这个值来确定保存数据的大小。

close() 方法，资源的 close 处理逻辑，同样由子类实现。

mark(int readlimit) 方法加了 synchronized 关键字，是一个对象级别的线程安全方法。该方法的定义规定调用该方法时，会标记当前 inputStream 流的位置，在后续调用 reset() 方法之后，可以返回当前标记的位置进行重新读取。reset() 同样也是线程安全方法，有子类实现。markSupported() 方法标记当前 InputStream 是否支持标记。
